The M Computer Language

Last updated 17 July 2021

This document and the software it describes is a work in progress. The software is not ready for use, and features described here may not be implemented yet.

 

M is language targeted at embedded systems. The name "M" comes from "embedded".

M will always be open.  The specification and any source code to implement it will be available for all to freely view, and to copy and use with minor restriction.  The copy restrictions will be intended to serve two purposes:

  1. To make sure the author is credited when any code is used. 
  2. To make sure the M specification is adhered to.  This prohibits a bunch of different M flavors out there confusing things.  You can still use the code to implement other languages or any other software, but you can't call the result M or claim it to be M-like. 

There will be no restriction on commercial use, or any onerous requirements such as that code added by others be made publicly available.  M becomes more valuable to all the more it is available and "out there".  Commercial implementations serve this purpose, so the copy restrictions are intended not to hinder them.
