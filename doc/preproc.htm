The M Computer Language
Preprocessor

Last updated 30 Jul 2023

M source files are actually translated by the Embed scripting language interpreter (ESCR) in preprocessor mode, with the result compiled as M. See the ESCR documentation for a description of how to use the preprocessor.

The specific configurations of the ESCR script processor in the M compiler are:

  1. The script processor is run in preprocessor mode, not script mode.
  2. Preprocessor comments start with "//" and extend to the end of the line. These comments are not transferred to the resulting M source.
  3. M comments are understood by the preprocessor as data file comments.
  4. Preprocessor commands start with "#".
  5. No additional preprocessor commands beyond the default set are added.
  6. Preprocessor functions start with "{" and end with "}".
  7. No additional preprocessor functions beyond the default set are added.
  8. Literal strings to the preprocessor are enclosed in quotes ("...").
  9. Include file names are assumed to end with ".ins.m".
