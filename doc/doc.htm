Documentation Convensions

Last updated 17 July 2021

This section describes how source code, key words, etc, are distinguised from each other and general text in the language descriptions. 

Keywords

M language keywords are shown in

bold face

M keywords are always lower case.  Keywords must be written exactly as shown. 

Description

Brief descriptions of what to write instead of the actual text to write are shown in

italic type

Such descriptions must be replaced by the item they describe in the actual source code. 

Source code

Verbatim source code is shown in 

fixed-space font

just like you would see it in a source code editor.
